Day 4 - Wednesday, March 1

7:29 am

Here is what I did as a first attempt at setting up a shared screen session:

  1. Logged into rteitelbaum, the laptop connected to the projector in the front of the classroom. Started an X terminal session and sshed from there to the ebgranville classroom web server.
  2. Logged in (using ssh) to ebgranville from my desktop workstation, ghopper, and ran the following commands: 
      $ sudo chmod u+s $(which screen)
  $ sudo chmod 755 /var/run/screen
  3. Confirmed that my .screenrc file had the following: 
      multiuser on

  # Grant users access to screen sessions
  acladd [username]
  ...
  acladd [username]
    and that my .bash_aliases file contained the following: 
      # screen aliases
  screens() { screen -aAOS $1;}
  4. Typed: 
      $ screens sharedscreen1
    in the projected virtual console.

Marco's instructions don't explain how users connect to a screen session, so I'll have to wait for him to read this and revise his instructions in response before testing further.
