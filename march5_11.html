Day 3 - Tuesday, March 7

Setting Up a novaweb Machine

Since it lacks a requirements.txt file or a virtualenv, and since both Flask and MySQL have changed significantly over the past two years, here is what I did to start trying to get the novaweb application running again:

  1. Setup a virtual machine (using KVM) running Ubuntu 14.04 server.
  2. Ran:
      $ sudo apt install mysql-server python-flask python-flask-login git
  3. Ran:
      git clone git@github.com:novawebdevelopment/novaweb.git

I set the MySQL root user password to novaweb. I'm taking this approach since the application was created two years ago, and it was either running on Debian or Ubuntu (probably, Debian, now that I think about it, but this is approach will probably have similiar enough versions of the required software). I want to avoid changes in the module import behavior of Flask, so I'm hoping the version installed from the distribution repository will work with the application.

After following the process described above and launching runserver.py, I encountered the error with a missing config.py file. That makes sense, since configuration information shouldn't be kept in a source repository, but now I need to figure out how to recreate the file. Marco told me that we have a version of this application on our WebFaction account. I'll ask him about that when he comes in today.
